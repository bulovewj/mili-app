> **현재 상태 (v1.0.0)**: 배포 완료  
> Netlify: `https://shimmering-biscochitos-0d662a.netlify.app`

> 서버리스 함수: `netlify/functions/grade.js` (Claude API 프록시)

## 0. 개발 개요

| 항목 | 내용 |
|------|------|
| 앱 이름 | 미리(MIRI) |
| 대상 | 초등 5~6학년 |
| 접속 환경 | 스마트폰·태블릿 브라우저 (모바일 퍼스트) |
| 구현 방식 | **단일 HTML 파일** (index.html 1개, 설치 불필요) |
| AI 연동 | Claude API (Anthropic) — 루브릭 기반 서술형 채점 |
| 데이터 저장 | localStorage (서버 없음, 기기 내 저장) |
| 언어 | 한국어 전용 |
| 배포 | Netlify (HTML 파일 드래그앤드롭 업로드) |

## 1. 디자인 원칙

- **분위기**: 귀엽고 밝은 느낌. 초등학생이 친근감을 느끼는 컬러와 폰트
- **모바일 퍼스트**: 기준 뷰포트 390px (iPhone 기준), 태블릿(768px)에서도 자연스럽게 확장
- **폰트**: 귀엽고 가독성 좋은 한국어 웹폰트 사용 (예: Noto Sans KR, 나눔고딕 등)
- **컬러**: 밝고 생동감 있는 파스텔 계열 메인 컬러, 활동별 구분 색상 사용
- **아이콘**: 이모지 또는 SVG 아이콘으로 각 활동 시각화
- **버튼**: 크고 탭하기 쉬운 터치 친화적 UI (최소 높이 56px)
- **애니메이션**: 화면 전환 시 부드러운 페이드/슬라이드 효과

### Screen 3 — Module 01 진단 퀴즈

**목적**: 학생의 초기 비판적 미디어 리터러시 수준 파악

**문항 구성** (4문항, 객관식 4지선다):

**Q1. 클릭베이트 식별**
> 다음 유튜브 썸네일 제목 중 클릭베이트일 가능성이 가장 높은 것은?
- ① "과학자들이 발견한 새로운 공룡 화석"
- ② "이걸 보면 절대 안 믿겠지만...충격 실화 (100% 사실)"
- ③ "우리 학교 급식 오늘 메뉴 소개"
- ④ "태양계 행성의 크기 비교"
- **정답: ②**

**Q2. 편향성 인식**
> 같은 사건을 다룬 두 기사가 서로 다른 내용을 강조한다면, 그 이유로 가장 적절한 것은?
- ① 기자가 실수로 잘못 썼기 때문에
- ② 기사를 쓴 매체마다 관점과 입장이 다를 수 있기 때문에
- ③ 한쪽 기사는 반드시 거짓이기 때문에
- ④ 사건이 두 가지로 일어났기 때문에
- **정답: ②**

**Q3. 광고·협찬 식별**
> 유튜버가 영상에서 특정 제품을 소개하며 "이 영상은 ○○브랜드의 협찬을 받았습니다"라고 밝혔다. 이 영상을 볼 때 어떤 태도가 가장 좋을까?
- ① 협찬을 받았으니 내용이 모두 거짓이다
- ② 협찬 사실과 관계없이 내용을 그대로 믿는다
- ③ 협찬을 받았으므로 제품에 유리한 내용일 수 있음을 고려하며 본다
- ④ 협찬 영상은 보지 않는 것이 좋다
- **정답: ③**

**Q4. 허위정보 판별**
> 친구가 단체 채팅방에 "○○ 먹으면 코로나 낫는다!"는 메시지를 보냈다. 가장 올바른 행동은?
- ① 나도 빠르게 다른 친구들에게 전달한다
- ② 재미있으니까 그냥 웃고 넘긴다
- ③ 공식 기관(질병청, 뉴스 등)에서 확인한 후 판단한다
- ④ 친구를 믿으니 사실이라고 생각한다
- **정답: ③**

**채점 방식**:
- 정해진 정답 기준으로 자동 채점 (AI 불필요)
- 4점 만점 (문항당 1점)
- 결과: 0~1점 → 하 / 2~3점 → 중 / 4점 → 상

**UI**:
- 한 번에 1문항씩 표시 (페이지네이션)
- 상단 진행 바 (1/4, 2/4...)
- 선택지 탭하면 즉시 선택 표시, "다음" 버튼으로 이동
- 이전 문항으로 돌아가기 불가 (단방향)

### Screen 5 — 홈 런처

- 상단: "{이름}의 미션 현황"
- 전체 진행률 표시 (완료한 활동 수 / 5)
- 5개 활동 카드 세로 목록 또는 2열 그리드

**활동 카드 구성**:

| 번호 | 활동명 | 아이콘 | 색상 |
|------|--------|--------|------|
| 1 | 유튜브 탐정 | 📺 | 빨강 계열 |
| 2 | 뉴스 비교 | 📰 | 파랑 계열 |
| 3 | 광고 탐정 | 🔍 | 노랑 계열 |
| 4 | AI 탐정 | 🤖 | 보라 계열 |
| 5 | 카드뉴스 | ✏️ | 초록 계열 |

- 완료한 활동: 체크 표시 + 획득 등급(상/중/하) 표시, 흐릿하게 처리
- 미완료 활동: 밝게 표시, 탭 가능
- 3개 이상 완료 시: 하단에 "역량 지형도 보기" 버튼 활성화

### Screen 6 — 유튜브 탐정 (클릭베이트 식별)

- 이미지: 행성 일러스트 (CSS로 구현)

썸네일 B (클릭베이트):
- 제목: "이거 보면 진짜 충격... 학교에서 절대 안 알려주는 비밀 (100% 실화)"
- 채널: 충격영상모음 · 조회수 987만
- 이미지: 느낌표 가득한 배경 (CSS로 구현)

썸네일 C (정상):
- 제목: "우리나라 전통 놀이 종류와 방법 소개"
- 채널: 문화유산채널 · 조회수 3.2만
- 이미지: 전통놀이 일러스트 (CSS로 구현)

**학생 활동**:
1. 클릭베이트라고 생각하는 썸네일 선택 (탭)
2. 서술형 이유 입력: "왜 클릭베이트라고 생각했어? 이유를 써봐!" (최소 10자)
3. "제출하기" 버튼 → AI 채점 화면으로

### Screen 7 — 뉴스 비교 (편향성 분석)

**시나리오**: 실제 뉴스 앱과 유사한 UI로 같은 사건의 두 기사 나란히 제시

**UI 재현 요소**:
- 뉴스 앱 스타일 상단 바 (가짜 로고)
- 두 기사 카드 나란히 (또는 탭 전환)
- 탐정 미션 안내: "같은 사건, 다른 기사! 차이를 찾아라!"

**콘텐츠 (하드코딩)**:

주제: 학교 급식 채식 선택제 도입

기사 A (긍정적 관점):
- 헤드라인: "학교 급식에 채식 선택제 도입, 학생 건강과 환경 모두 잡는다"
- 본문 요약: "전문가들은 채식 급식이 건강에 유익하며 탄소 배출 감소에 기여한다고 밝혔다. 시범학교 학생들의 만족도는 78%로 높게 나타났다."
- 출처: 건강한급식신문

기사 B (부정적 관점):
- 헤드라인: "채식 급식 강요 논란... '먹고 싶은 것도 못 먹나' 학생 불만"
- 본문 요약: "일부 학생들은 '반찬 선택이 줄었다'며 불만을 토로했다. 학부모들도 영양 불균형을 우려하는 목소리가 높아지고 있다."
- 출처: 학부모뉴스

**학생 활동**:
1. 두 기사를 읽은 후 서술형 입력
2. 질문: "두 기사는 어떻게 다르게 보도하고 있어? 왜 그런 차이가 생겼을까?" (최소 15자)
3. "제출하기" 버튼 → AI 채점 화면으로

### Screen 9 — AI 탐정 (허위조작정보 식별)

**시나리오**: 카카오톡 단체채팅방 UI 재현, 메시지 중 허위정보 탐지

**UI 재현 요소**:
- 카카오톡 스타일 채팅창 (가짜)
- 상단: "6학년 1반 단톡방" 채팅방 이름
- 여러 메시지 말풍선

**콘텐츠 (하드코딩)**:

메시지 흐름:
1. 민준: "얘들아 내일 학교 안 온대! 태풍 온다고"
2. 수아: "진짜? 어디서 봤어?"
3. 민준: "인스타에서 봤어 ㅋㅋ 공유 엄청 됐던데"
4. 지호: "🚨속보🚨 내일 전국 모든 학교 휴교!! 기상청 발표 (사진첨부)" ← 이상한 메시지
5. 사진: 뉴스처럼 보이지만 출처 불명확한 이미지 (CSS로 구현, 글씨 흐릿하게)
6. 하은: "와 진짜? 신난다!!"

**학생 활동**:
1. 의심스러운 메시지/정보를 보고 서술형 입력
2. 질문: "이 채팅방에서 가짜 정보일 수 있는 게 뭐야? 왜 의심스러워?" (최소 10자)
3. "제출하기" 버튼 → AI 채점 화면으로

> **주의**: 정확한 식별이 목표가 아님. "의심하는 태도 + 왜곡 유형 언어화"를 평가함.

### Screen 10 — 카드뉴스 만들기 (비판적 표현)

**시나리오**: 직접 카드뉴스를 만들어보며 비판적 표현 역량 발휘

**UI**:
- 카드뉴스 편집 화면 스타일
- 주제 고정: "지구 환경을 지키자" (학생이 바꿀 수 없음)
- 학생 입력 영역:

AI 탐정:

중(65점): 클릭베이트 없으나 막연한 제목 / 막연한 응원·경각심 / 이미지 선택했으나 연결 설명 없음

## 7. 개발 시 주의사항

1. **단일 파일**: 모든 HTML, CSS, JS를 index.html 1개에 구현
2. **외부 의존성 최소화**: CDN은 구글 폰트만 허용, 나머지는 순수 JS
3. **API 키**: 실제 키는 Netlify 환경변수 `ANTHROPIC_API_KEY`에만 저장, 브라우저 코드에 절대 포함 금지
4. **오프라인 대응**: API 호출 실패 시 앱이 멈추지 않고 에러 메시지만 표시
5. **모바일 최적화**: viewport meta 태그 필수, 터치 이벤트 처리, 스크롤 자연스럽게
6. **한국어 전용**: 모든 텍스트, 안내문, 버튼 한국어로
7. **이미지**: 외부 이미지 URL 사용 금지, CSS/SVG로 모든 시각 요소 구현
8. **레이더 차트**: 외부 라이브러리 없이 Canvas API 또는 SVG로 직접 구현

## 9. API 보안 규칙

### 핵심 원칙

실제 API 키를 프론트엔드 코드에 절대 넣지 않는다.

아래 위치는 모두 공개 접근 가능하므로 시크릿 보관 금지:
- `index.html`, 인라인 JS, 번들 JS
- `localStorage`, `sessionStorage`, URL/쿼리스트링
- HTML 주석, 공개 Git 저장소

### 보안 아키텍처

```
브라우저(index.html)
  → POST /.netlify/functions/grade
       { activityId, studentAnswer }
  → netlify/functions/grade.js
       (API 키 읽기, 루브릭 보관, Claude 호출)
  → api.anthropic.com/v1/messages
```

브라우저 코드에서 `https://api.anthropic.com`을 직접 호출하는 패턴은 절대 사용 금지.

### 서버리스 함수 필수 검증 항목

```javascript
// grade.js 에서 반드시 검증
- activityId 허용 목록: ['youtube', 'news', 'ad', 'ai', 'cardnews', 'summary']
- studentAnswer 길이 제한: 최대 1000자
- 빈 답변 거부
- 모델명 서버 고정: claude-sonnet-4-5
- max_tokens 서버 고정: 채점 300, 요약 400
- temperature 서버 고정: 0
```

### 환경변수 설정 (Netlify)

```
ANTHROPIC_API_KEY=sk-ant-...   ← Netlify 대시보드에만 입력, 코드에 절대 금지
ALLOWED_ORIGIN=https://shimmering-biscochitos-0d662a.netlify.app
```

### API 키가 노출된 경우

1. 즉시 Anthropic 대시보드에서 해당 키 폐기
2. 새 키 발급 → Netlify 환경변수에만 저장
3. 노출된 키가 포함된 커밋 히스토리 정리
4. 사용 로그 확인

### 학생 데이터 프라이버시

- Claude API에는 답변 텍스트만 전송 (이름 포함 금지)
- 채점 결과는 localStorage에만 저장 (서버 전송 없음)
- 서버 프로덕션 로그에 학생 답변 내용 기록 금지
