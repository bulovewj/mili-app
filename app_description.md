# 미리(MIRI) 앱 설명서

> 초등 5~6학년 대상 AI 미디어 리터러시 교육 앱  
> 작성 기준: v1.0.0 (2026-06-01 현재 배포 버전)

---

## 1. 앱 개요

| 항목 | 내용 |
|------|------|
| 앱 이름 | 미리(MIRI) — 미디어 탐정 |
| 대상 | 초등 5~6학년 |
| 접속 방법 | URL 하나로 브라우저에서 바로 접속 (설치 불필요) |
| 배포 주소 | https://shimmering-biscochitos-0d662a.netlify.app |
| 파일 구조 | 단일 HTML 파일 (`index.html`) + Netlify 서버리스 함수 |
| 데이터 저장 | localStorage (기기 내 저장, 서버 없음) |
| AI 연동 | Anthropic Claude API (서버리스 함수 경유, 서술형 채점용) |

### 학습 목표
학생이 실제 미디어 환경(유튜브, 뉴스, SNS, 카카오톡, 카드뉴스)을 시뮬레이션하며 다음 역량을 기름:
- 클릭베이트 탐지
- 뉴스 편향성 분석
- 광고·협찬 식별
- 허위정보 탐지
- 비판적 표현 능력

## 2. 전체 화면 흐름

```
[시작 화면] 이름 입력
     ↓
[앱 소개] 5가지 미션 미리보기
     ↓
[진단 퀴즈] 4문항 객관식 → 초기 수준 파악
     ↓
[진단 결과] 레벨 확인 (탐정 견습생 / 탐정 / 수석 탐정)
     ↓
[홈 런처] 5개 활동 카드 목록
     ↓ (각 활동 선택)
[활동 1~5] 각 활동 수행 → 서술형 제출
     ↓
[채점 결과] PASS/FAIL + AI 피드백
     ↓ (3개 이상 완료 시)
[역량 지형도] 레이더 차트 + AI 종합 피드백
```

## 3. 화면별 상세 설명

### 화면 1 — 시작 화면 (`screen-start`)

- 앱 로고(돋보기+영상 SVG)와 "미리(MIRI)" 제목 표시
- 이름 입력 필드 (최대 10자, 닉네임 가능)
- 이름 입력 시 "시작하기" 버튼 활성화
- 새로고침·첫 접속 시 항상 이 화면부터 시작 (localStorage에 이름이 있어도)
- 입력한 이름은 localStorage에 저장, 이후 전체 앱에서 "{이름}아/야"로 호칭

---

### 화면 2 — 앱 소개 (`screen-intro`)

- "{이름}아/야, 반가워! 👋" 인사 (받침 유무 자동 처리)
- 앱 설명 2줄 + 5개 활동 아이콘(📺📰🔍🤖✏️) 미리보기
- "미션 시작! 🎯" 버튼 → 진단 퀴즈로 이동

---

### 화면 3 — 진단 퀴즈 (`screen-diagnosis`)

한 번에 1문항씩 표시. 상단 진행 점 표시(●○○○). 정답 선택 즉시 피드백, 일부 문항은 개념 설명 박스 표시.

**4문항 구성:**

| 번호 | 주제 | 정답 |
|------|------|------|
| Q1 | 정보 확인 방법 — 공유 횟수가 많으면 믿을 수 있다? | ② (옳지 않은 것) |
| Q2 | 클릭베이트 식별 — 자극적 제목 영상 대처 방법 | ④ 무조건 믿지 않는다 |
| Q3 | 뉴스 비판적 읽기 — 구독자 많은 유튜버 말은 믿어도 된다? | ③ (옳지 않은 것) |
| Q4 | 광고·협찬 — 좋아하는 유튜버 제품 추천 대처 | ③ 협찬 여부 확인 후 비판적으로 |

- Q2, Q3, Q4는 정답 선택 후 개념 설명 박스(클릭베이트·편향성·광고협찬 설명) 표시
- 채점: 4점 만점 → 4점=상(🏆 수석 탐정), 2~3점=중(🕵️ 탐정), 0~1점=하(🔰 탐정 견습생)

---

### 화면 4 — 진단 결과 (`screen-diagnosis-result`)

- 등급 이모지 + 등급명 + 정답 수 표시
- 등급별 격려 메시지
- "5가지 미션 도전하기!" → 홈 런처로 이동

---

### 화면 5 — 홈 런처 (`screen-home`)

- "{이름}의 미션 현황 🗂️" + 전체 진행률 바 (완료 수/5)
- 5개 활동 카드 세로 목록
  - 완료 시: 카드 흐릿하게 + PASS/FAIL 뱃지 표시
  - 이미 완료한 활동 탭 시: "결과 다시 보기 / 다시 도전하기 / 취소" 모달 팝업
- 3개 이상 완료 시 "🗺️ 역량 지형도 보기" 버튼 활성화

### 화면 6 — 유튜브 탐정 (`screen-youtube`)

**학습 역량:** 클릭베이트 탐지

실제 유튜브 UI 재현 (로고, 검색 아이콘 포함). 썸네일 3개를 무작위 순서로 표시.

**3개 썸네일 내용:**

| ID | 채널 | 제목 | 유형 |
|----|------|------|------|
| A | KBS 뉴스 | [단독] 교육부, 초등학교 스마트폰 수업 중 사용 전면 금지 추진 | 정상 뉴스 |
| B | 교육부 공식채널 | 학교 스마트폰 사용 어떻게 바뀌나? | 정상 공식 |
| C | 어린이크리에이터_뚜뚜 | ⚡무조건 시청!! 교실 스마트폰 전면 허용?! 안보면 손해!! 🔥 (충격 실화) | **클릭베이트** |

**학생 활동:**
1. 클릭베이트라고 생각하는 썸네일 탭하여 선택
2. 서술형 이유 입력 (최소 10자)
3. "탐정 보고서 제출하기!" → Claude API 채점

**채점 방식 (PASS ≥ 50점):**
- 정답(C 선택): +30점
- 서술형 답변: AI 루브릭 채점 (최대 70점)
  - 하(0점): 이유 없거나 1~2단어
  - 중(35점): 클릭베이트 특징 1가지
  - 상(70점): 특징 2가지 이상 또는 정상 채널과 비교

### 화면 7 — 뉴스 비교 (`screen-news`)

**학습 역량:** 뉴스 편향성 분석

실제 뉴스 앱 UI 재현 (탭 전환 방식). 주제: **AI 검색 도입**.

**두 기사 내용:**

| | 기사 A | 기사 B |
|-|----|-----|
| 매체 | 🔵 디지털교육신문 | 🔴 책사랑뉴스 |
| 헤드라인 | "AI 검색으로 궁금증 바로 해결, 학생 학습 효율 높아졌다" | "AI만 찾는 아이들, 도서관 텅텅… 깊이 있는 사고력 잃어가" |
| 관점 | 긍정적 (74% 만족도, 교사 호평) | 부정적 (도서관 41% 감소, 사고력 우려) |

**3단계 질문:**

- **Q1 (객관식):** "기사에 나온 내용은 모두 사실일까?" → 정답: 기자의 의견도 포함됨 (자동 채점)
- **Q2 (선택형, 각각):** 두 기사에서 의견이 드러난 문장 고르기 (각 3개 선택지)
  - 기사 A 정답: ③ "이를 적극 활용하도록 돕는 것이 교육의 방향이 되어야 한다"
  - 기사 B 정답: ③ "책을 통해 스스로 생각하는 힘을 잃어가고 있는 것은 아닌지 깊이 우려된다"
- **Q3 (서술형):** "같은 '기사'인데 왜 두 기사의 관점이 서로 다를까?" (힌트: 기사를 쓴 사람은 모두 같은 생각을 가지고 있을까요?)

**채점 방식:**
- Q1 정답: 20점
- Q2A + Q2B 각 50점 × 0.4 = 최대 40점
- Q3 서술형 AI 채점 × 0.4 = 최대 40점 (카테고리 2개↑=100점, 1개=50점, 0개=0점)
- 합산 최대 100점, PASS ≥ 50점

### 화면 8 — 광고 탐정 (`screen-ad`)

**학습 역량:** 광고·협찬 식별

실제 SNS 피드 UI 재현 (인스타그램 스타일).

**3개 게시물:**

| 게시물 | 계정 | 내용 | 유형 |
|--------|------|------|------|
| A | 초6_김도윤 | 체육시간 피구 에피소드 | 일반 게시물 |
| B | 키즈유튜버_나연 | 🏷️ [협찬] 색연필 추천 | **명시적 광고** |
| C | 테크리뷰_민서 | (광고) 갤럭시 탭 자연스러운 추천 | **광고 표시 있는 광고** |

**3단계 질문:**

- **Q1 (복수선택):** 광고·협찬 게시물 모두 고르기 → 정답: B, C 모두 선택
- **Q2 (객관식):** "게시물 C는 (광고)라고 적혔는데도 광고처럼 안 느껴져. 왜?" → 정답: 자연스러운 일상 말투
- **Q3 (서술형):** "광고인지 드러내지 않고 제품을 광고하는 것을 '뒷광고'라고 해. 이런 뒷광고에는 어떤 문제가 생길 수 있을까?" (최소 10자)

**채점 방식:**
- Q1 정답: 20점 (B, C 모두 선택 시)
- Q2 정답: 20점
- Q3 서술형 AI 채점 × 0.6 = 최대 60점
- 합산 최대 100점, PASS ≥ 50점

### 화면 9 — AI 탐정 (`screen-ai`)

실제 카카오톡 UI 재현 (노란 헤더, 파란 채팅창). 채팅방 이름: "6학년 1반 단톡방"

**채팅 내용:**

| 발언자 | 시간 | 내용 |
|--------|------|------|
| 수아 | 2:14 | "얘들아 내일 태풍 엄청 온다는데 혹시 학교 못가는 거 아냐?" |
| 민준 | 2:14 | "에이 설마ㅋㅋ 아직 뉴스나 선생님 말씀이 없었는데?" |
| 다예 | 2:15 | "인스타에서는 본 것 같기도 한데.." |
| 지호 | 2:15 | 🚨속보🚨 여기서 학교 안간대! 내일 논다!! *(의심 말풍선 + 가짜 뉴스 카드: 출처 불명, 공식 기관 미확인)* |

- **Q2 (객관식):** "지호의 '속보' 메시지를 의심해야 하는 이유로 **옳지 않은 것**은?" → 정답: ② 공식 기관에서 직접 확인된 내용이다
- **Q3 (서술형):** "단톡방에서 이런 메시지를 받으면 어떻게 해야 할까? 내가 할 행동을 써봐!" (최소 10자)

**채점 방식:**
- Q1 정답: 20점
- Q2 정답: 20점
- Q3 서술형 AI 채점 × 0.6 = 최대 60점
- PASS ≥ 50점

### 화면 10 — 카드뉴스 만들기 (`screen-cardnews`)

**학습 역량:** 비판적 표현 능력

주제 고정: **북극 환경**. 작성 전 참고 자료 3개 제공 (북극곰 개체수, 전문가 발언, 해빙 면적 수치).

**4단계 입력:**

1. **제목 선택** — 3개 중 1개 선택:
   - A (클릭베이트): "10년 뒤 북극곰 전부 멸종한다고?? (충격 실화🐻‍❄️)"
   - B (클릭베이트): "지구가 곧 끝난다!! 환경 전문가들도 포기한 충격적인 진실"
   - C (정답): "북극 해빙 면적, 40년 만에 절반으로 줄었어요"
   - A·B 선택 시 오답 피드백 + 재선택 가능, C 선택 시 나머지 폼 공개

2. **이미지 선택** — 실제 이미지 파일 3개 중 1개 탭 선택 (북극곰 / 북극 얼음 / 지구환경)

3. **핵심 메시지** — 자유 입력 (최대 100자, 수치·사실 포함 권장)

4. **이미지 선택 이유** — 자유 입력 (최소 10자)

**채점 방식:**
- 제목 정답(C) 선택: 20점
- 서술형(메시지+이유) AI 채점 = 최대 80점
- PASS ≥ 50점

---

### 화면 11 — AI 채점 결과 (`screen-result`)

모든 활동 완료 후 이 화면으로 이동.

- **PASS** (✅, 초록) / **FAIL** (❌, 빨강) + 점수
- AI 피드백 텍스트 (Claude가 생성)
- FAIL 시 "다시 도전하기" 버튼 표시
- "홈으로 돌아가기" → 홈 런처로 복귀, 활동 완료 처리

---

### 화면 12 — 역량 지형도 (`screen-radar`)

3개 이상 활동 완료 시 홈에서 접근 가능.

- **레이더 차트**: Canvas API로 직접 구현 (5각형, 5개 꼭짓점)
  - 각 꼭짓점: 클릭베이트 탐지 / 편향성 분석 / 광고 식별 / 허위정보 탐지 / 비판적 표현
  - 미완료 활동은 회색 처리
- **활동별 점수 목록**: PASS/점수 표시
- **AI 종합 피드백**: 완료한 활동 데이터 기반 Claude가 맞춤 생성
- "처음부터 다시 도전하기" → localStorage 초기화 후 시작 화면

## 4. AI 채점 시스템

### 구조

```
브라우저(index.html)
  → POST /.netlify/functions/grade
      { activityId, studentAnswer }
  → netlify/functions/grade.js
      (Claude API 호출, 루브릭 적용)
  → 결과 반환: { score, pass, feedback, ... }
```

- **API 키**: Netlify 환경변수 `ANTHROPIC_API_KEY`에만 저장 (브라우저 코드에 절대 없음)
- **모델**: `claude-sonnet-4-5` (grade.js에 고정)
- **temperature**: 0 (채점 일관성)

### 활동별 AI 채점 루브릭

| 활동 | 키워드 카테고리 | 100점 | 50점 | 0점 |
|------|--------------|-------|------|-----|
| 유튜브 탐정 | 제목 과장/자극성, 내용과 제목 불일치, 조회수 신뢰 불가 | 2개↑ | 1개 | 0개 |
| 뉴스 비교 Q3 | 관점·입장·의도, 매체마다 다르다, 기자 생각·가치관 | 2개↑ | 1개 | 0개 |
| 광고 탐정 Q3 | 소비자 오인, 신뢰 훼손, 허위 정보, 공정거래 위반 | 2개↑ | 1개 | 0개 |
| AI 탐정 Q3 | 출처 확인, 공식 기관 확인, 공유 전 검증, 전달 자제 | 2개↑ | 1개 | 0개 |
| 카드뉴스 | 클릭베이트 없음, 구체적 사실·수치, 이미지-메시지 연결 | 2개↑ | 1개 | 0개 |
| 역량 지형도 | — | 종합 피드백 자유 생성 | — | — |

### PASS/FAIL 기준

전 활동 공통: **50점 이상 = PASS**, 49점 이하 = FAIL

## 5. 데이터 저장 구조 (localStorage)
